## Cloud Reviewer Agent

A new specialist agent has been added at `.github/agents/cloud-reviewer.agent.md` that reviews code and infrastructure for **cloud deployment readiness**.

### When to Invoke

Trigger phrases: `review for cloud`, `cloud readiness`, `review infrastructure`, `check IaC`, `cloud security review`, `review before deploy`, `pre-deployment check`, `cloud audit`.

### What It Reviews (5-Step Process)

| Step | Focus | Sources |
|------|-------|---------|
| 0 | Context analysis — system type, cloud provider, review depth | |
| 1 | Security — OWASP Top 10, LLM Top 10, secrets detection, IAM least-privilege | `se-security-reviewer` |
| 2 | IaC — Terraform/CDK/CloudFormation state safety, drift, module design | `terraform-iac-reviewer` |
| 3 | Architecture — Well-Architected pillars (reliability, cost, performance, ops) | `se-system-architecture-reviewer` |
| 4 | AI/Agent governance — prompt injection, trust scoring, audit trails (LLM systems only) | `agent-governance-reviewer` |
| 5 | Output — Markdown report saved to `docs/cloud-reviews/[date]-[component]-review.md` | |

### Review Depth

| Depth | When to Use |
|-------|-------------|
| `full` | Pre-production, security-sensitive, AI/LLM systems — all 5 steps |
| `standard` | Feature PRs, infra changes — steps 1–3 |
| `lightweight` | Hotfixes, minor config changes — step 1 (security/secrets) only |

### Priority Levels

- 🔴 **CRITICAL** — Block deploy (exposed secrets, broken auth, state corruption, wildcard IAM)
- 🟡 **IMPORTANT** — Fix before production (no rate limits, no DR plan, unencrypted storage)
- 🟢 **SUGGESTION** — Non-blocking improvement (cost optimization, tagging conventions)

### Integration with Dev Quality Lead

The `cloud-reviewer` agent can be invoked as part of the `dev-quality-lead` gate sequence before any cloud infrastructure deployment, in addition to the existing code quality and test gates.
