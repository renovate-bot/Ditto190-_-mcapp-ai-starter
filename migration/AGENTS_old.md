# Agents in this repository

This repository includes workspace-scoped Copilot Chat customizations to improve in-chat assistance.

- `.github/agents/copilot-chat.agent.md` — project-scoped agent definition and discovery triggers.
- `.github/prompts/copilot-chat.prompt.md` — workspace prompt template used by the agent.
- `.github/instructions/copilot-instructions.md` — always-on repository instructions and conventions.

How to use

1. Install and enable the GitHub Copilot Chat extension in VS Code.
2. Open Copilot Chat and start a new conversation. The workspace instructions and agents should be discoverable via trigger phrases in the agent frontmatter `description`.
3. To customize triggers or scope, edit the frontmatter `description` or `applyTo` fields.

If you'd like, I can also add examples, more granular `applyTo` globs, or move these files to the `.agents/` or `.github/` locations you prefer.

Prompt examples

- "Search awesome-copilot for 'repo architect' and show top 5 matches"
- "Load 'custom-agent-foundry.agent.md' and display its frontmatter/tools for review"
- "Find agents for 'security review' and list recommended triggers/prompts"

This project now tracks loaded/copied agent assets in a local registry so they can be re-indexed and regenerated.

- Registry cache root: `.agents/registry/awesome/`
- Latest index snapshot: `.agents/index/agent-assets.index.json`
- Index build log: `.agents/index/agent-assets.log.jsonl`
- Awesome load log: `.agents/index/awesome-loads.jsonl`

Pipeline scripts:

- `scripts/build_agent_index.py` — scans `.github` + `.agents` asset folders and writes index/log files.
- `scripts/register_awesome_asset.py` — registers a loaded awesome-copilot artifact into `.agents/registry/awesome/...` and appends to load log.
- `scripts/watch_agent_index.ps1` — watches agent-related folders and auto-runs index build on file changes.
- `scripts/regenerate_from_registry.py` — restores assets into workspace config folders from logged registry entries.

## Wasp CLI installation: GitHub Codespaces & Linux VMs

This section describes a reproducible configuration to make the `wasp` CLI available in two environments commonly used for development: GitHub Codespaces (which uses Linux containers/VMs) and general Linux virtual machines.

Files added to this repo to help automation:

- `./.devcontainer/devcontainer.json` — Codespaces / Devcontainer configuration that installs Node and runs the project install helper after container creation.
- `./scripts/install-wasp-linux.sh` — Idempotent installer script for Linux environments (Codespaces, cloud VMs) that ensures Node >= 22.22.2 and installs `@wasp.sh/wasp-cli` globally, then verifies `wasp --version`.

Key constraints & requirements

- Wasp CLI (official package `@wasp.sh/wasp-cli`) currently supports Linux and macOS; Windows native npm install is not supported. For Windows developers, use Codespaces, WSL, or a Linux VM.
- Wasp requires Node.js >= v22.22.2. The provided installer uses `nvm` to ensure the correct Node version when needed.

Codespaces / Devcontainer setup (recommended)

1. Ensure the repository contains `./.devcontainer/devcontainer.json` (added here). It uses an Ubuntu-based image and runs the install helper as `postCreateCommand` so the `wasp` CLI is available inside Codespaces and VS Code Remote - Containers.
2. Open the repository in a Codespace or via VS Code Remote - Containers. The container build will run `postCreateCommand` which executes `./scripts/install-wasp-linux.sh`.

Example `devcontainer.json` (this project includes `./.devcontainer/devcontainer.json`):

```json
{
  "name": "workspace-dev",
  "image": "mcr.microsoft.com/vscode/devcontainers/base:ubuntu",
  "features": {},
  "postCreateCommand": "bash /workspaces/${localWorkspaceFolderBasename}/scripts/install-wasp-linux.sh"
}
```

Linux VM / cloud VM setup

1. Copy `./scripts/install-wasp-linux.sh` to the target VM (or run it directly from the repo if mounted).
2. Run it as a non-root user (it will use `sudo` for global npm install if necessary):

```bash
bash ./scripts/install-wasp-linux.sh
# then verify
wasp --version
```

What the installer does (summary)

- Ensures `nvm` is installed and activates Node >= 22.22.2 (installs it if missing).
- Installs the official CLI package: `npm install -g @wasp.sh/wasp-cli@latest` (Linux-compatible package).
- Creates a small marker file on success and prints `wasp --version`.

Notes and troubleshooting

- If `wasp --version` still fails after running the script inside Codespaces, ensure you re-open the terminal inside the container or restart the Codespace — the PATH may need to be reloaded.
- On Linux VMs running under restricted images, you may need to run the installer with `--unsafe-perm` for `npm` global installs or run the script as root.
- For Windows dev machines, use WSL or Codespaces; see earlier notes in this document.

Paths of helper artifacts in this repo:

- `./.devcontainer/devcontainer.json` — devcontainer config for Codespaces
- `./scripts/install-wasp-linux.sh` — installer script for Linux environments
